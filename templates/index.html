Project Overview

This project aims to predict the churn risk of customers using machine learning techniques. The dataset contains customer-related information such as demographics, transaction details, and behavior patterns, which are used to estimate the likelihood of customer churn. The process involves data cleaning, preprocessing, visualization, and model training using different machine learning algorithms including Decision Tree, Random Forest, and an Artificial Neural Network (ANN).

1. Data Preprocessing

- Dataset Import: The data is imported from a CSV file (`train.csv`). Missing categorical values in columns like `region_category` are filled with the mode, and missing numerical values in `points_in_wallet` are filled with the mean.
- Feature Engineering: Date-related features are extracted from `joining_date` and `last_visit_time`, and one-hot encoding is applied to categorical columns. Outliers in `avg_transaction_value` are removed using the IQR method.

2. Data Visualization

- Histograms, correlation heatmaps, and boxplots were used to explore relationships and the distribution of features, with a special focus on `avg_transaction_value` and its relation to `churn_risk_score`.

3. Feature Selection

- Using an ExtraTreesRegressor, the top 20 important features for predicting churn risk were identified. These features include `age`, `avg_transaction_value`, and `membership_category`.

4. Model Training

- Three models were trained: Decision Tree, Random Forest, and an Artificial Neural Network (ANN). The Random Forest model was saved as `random_forest_model.pkl` and the ANN was built using PyTorch.

5. Model Evaluation

- Evaluation metrics were calculated for all models. Random Forest performed better than the Decision Tree, and the ANN further improved the predictions.

6. Model Inference

- Both the Random Forest and ANN models were used to make predictions on unseen data.
